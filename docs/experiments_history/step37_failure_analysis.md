## 🔍 1. 원인 분석 1: 학습 데이터의 구조적 왜곡 (데이터 진공, Data Vacuum)

성능 급락의 가장 결정적인 원인은 **이진 분류용으로 다운샘플링된 학습 데이터를 회귀 모델 학습에 그대로 재사용**한 것입니다.

### 1.1 분류용 다운샘플링과 회귀 모델의 부적합성
* **이진 분류(Step 36)**: Hit(1)와 Miss(0) 클래스 비율을 맞추기 위해, 1.0cm 이내인 `Hit` 후보 중 **가장 거리가 가까운 1개의 후보(`best_idx`)만 남기고** 나머지 `(best_idx, 1.0cm]` 구간의 우수 후보들을 학습 데이터셋에서 전부 삭제(Omit)했습니다.
* **회귀(Step 37)**: 회귀 모델은 실제 거리인 `reg_target`을 타깃으로 학습하지만, 제공된 학습 데이터에는 **0.4cm 이하의 초우수 후보 1개**와 **1.01cm 이상의 Near-Miss 후보들**만 존재했습니다. 즉, `0.5cm ~ 1.0cm` 구간의 데이터가 아예 비어 있는 **데이터 진공(Vacuum)** 상태였습니다.

### 1.2 테스트 추론 시의 파국적 영향
* 실제 테스트 세트 추론(Inference) 시에는 `0.5cm ~ 1.0cm` 사이의 모든 중간 거리를 가진 후보들이 입력됩니다.
* 하지만 모델은 학습 단계에서 이 구간의 오차 데이터를 전혀 보지 못했기 때문에(Out-of-Distribution), 이 후보들의 거리를 변별하지 못하고 **예측 거리값을 1.0cm 경계 부근으로 편향되게 예측**하여 1.0cm 경계선에 피크 오차가 걸리는 병목을 초래했습니다.
* **지표 대조 (FAST_TURNING)**:
  * **오라클 최선 오차(Oracle Best Median)**: **0.4380 cm** (후보군 중 물리적으로 정답에 가장 가까운 값)
  * **모델 실제 선택 오차(Selected Median)**: **1.0032 cm**
  * 모델이 충분히 정답 안쪽(0.44cm)에 있는 후보를 고를 수 있었음에도 불구하고, 학습 데이터 왜곡으로 인해 1.0cm 부근의 아슬아슬한 오답 후보를 선택하게 되었습니다.

## 🔍 2. 원인 분석 2: GMM 데이터 희소성(Sparsity) 하의 회귀 과적합 (Overfitting)

* **이진 분류 모델의 특징**: GBDT 의사결정나무는 분류 임계값(Threshold)을 기준으로 1.0cm 안팎을 가르는 경계선을 학습하는 것이 비교적 쉽습니다. 데이터가 적어도(예: SLOW_EXTREME_TURNING의 775개 ID) 경계를 잘 가릅니다.
* **회귀 모델의 특징**: 회귀 모델은 55개의 비선형 물리 피처 공간에서 **실제 센티미터 단위의 거리 함수 자체를 정교하게 근사**해야 합니다.
* **문제점**: GMM 분할로 인해 특정 Regime(선회)의 데이터 수($N$)가 지나치게 적은 상황에서 복잡한 MAE 거리 함수를 학습시키다 보니, 검증 셋(OOF)에서는 bagging을 통해 점수가 잘 나오는 것처럼 보였으나 리더보드 테스트 셋에서는 극심한 **과적합(Overfitting)과 예측 분산(Variance)**을 일으켰습니다.

## 🛠️ 3. 해결 방안 (Step 38 구현 핵심)

리더보드 Peak 스코어인 0.6728 구조를 보존하면서 회귀 모델의 거리 예측력을 극대화하기 위해 다음과 같은 설계 개편을 적용합니다.

### 3.1 균등 거리 빈 샘플링 (Uniform Distance Bin Sampling) 도입
* 학습 데이터 생성 스크립트([prepare_data.py](file:///d:/Repos/dacon-mosquito-trajectory/step38_2regime_regression/prepare_data.py))를 수정하여 오차 거리를 `[0~0.5cm]`, `[0.5~1.0cm]`, `[1.0~1.5cm]`, `[1.5~3.0cm]` 등 5개 구간으로 균등 분할합니다.
* 각 구간별로 후보를 6개씩 추출하여 데이터의 총 크기(ID당 25~30개)는 제한하되, **0.0cm부터 3.0cm까지의 거리 그라디언트를 고르게 복원**합니다.
* 이를 통해 모델이 1.0cm 이내의 모든 실수 거리 도메인을 완벽히 학습할 수 있게 만듭니다.

### 3.2 GMM 4-Regime 및 맞춤형 물리 격자 구조 유지
* Step 36의 0.6728 스코어 원동력이었던 **GMM 4개 Regime 분할 및 맞춤형 격자 생성 로직(physics.py)**은 그대로 상속하여 물리적 한계 커버리지(Overall 81.80%)를 보장합니다.
