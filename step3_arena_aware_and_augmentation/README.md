# Step 3: 아레나 인식 및 데이터 증강 (Arena-Aware & Augmentation)

Step 3에서는 모델의 예측 안정성을 높이기 위해 **공간적 컨텍스트(Arena Context)**를 도입하고, 데이터의 다양성을 확보하기 위한 **증강(Augmentation)** 기법을 적용합니다.

---

## 1. 주요 개선 사항

### 1.1. 아레나 인식 피처 (Arena-Aware Features)
단순히 모기의 움직임만 보는 것이 아니라, 현재 모기가 아레나 내의 **어디에 위치해 있는지**에 대한 정보를 모델에 추가로 제공합니다.
*   **벽면 거리 (6종)**: 현재 위치에서 상/하/좌/우/전/후 6개 벽면까지의 거리.
*   **절대 고도 (1종)**: 현재 모기의 절대적인 $Z$축 좌표.
*   **효과**: 모델이 "천장에 가까우면 더 이상 위로 날 수 없다"거나 "바닥 근처에서는 비행이 안정적이다"라는 물리적 제약 조건을 스스로 배울 수 있게 합니다.

### 1.2. 데이터 증강 (Data Augmentation)
*   **랜덤 회전 (Random Z-Rotation)**: 학습 시 궤적 데이터를 $Z$축을 중심으로 무작위로 회전시킵니다.
*   **효과**: 모기는 어느 방향으로든 날 수 있으므로, 특정 방향에 치우치지 않는 범용적인 비행 패턴을 학습하게 됩니다. (데이터 양이 수십 배로 늘어나는 효과)

---

## 2. 모델 아키텍처 (Hybrid Encoder)

Step 3의 모델은 두 개의 독립된 입력을 하나로 합치는 **하이브리드 구조**를 가집니다.
1.  **GRU (시계열 인코더)**: 과거 궤적의 '흐름'을 파악합니다.
2.  **MLP (공간 인코더)**: 현재 위치의 '아레나 맥락'을 파악합니다.
3.  **Concatenation**: 두 정보를 합쳐서 최종적으로 어떤 앵커(Anchor)를 선택할지, 그리고 미세 조정(Offset)은 얼마나 할지 결정합니다.

---

## 3. 학습 로직 설명 (`train.py`)

*   **손실 함수 (Loss Function)**:
    *   **Classification**: 6가지 앵커 중 정답과 가장 가까운 앵커를 맞추도록 학습 (CrossEntropy).
    *   **Regression**: 선택된 앵커로부터 정답까지의 오차를 줄이도록 학습 (SmoothL1Loss).
*   **SmoothL1Loss 도입**: 기존 MSE보다 노이즈(튀는 값)에 강한 손실 함수를 사용하여 학습의 안정성을 높였습니다.
*   **스케줄러 (StepLR)**: 학습 중반 이후에 학습률(Learning Rate)을 절반으로 낮추어, 더 세밀한 오차 보정이 가능하도록 설정했습니다.

---

## 4. 기대 효과
Step 2에서의 급격한 성능 저하 원인이었던 **'잘못된 의도 파악'** 문제를 아레나 컨텍스트와 데이터 증강으로 해결함으로써, 물리적으로 타당하고 강건한(Robust) 예측이 가능해질 것으로 기대합니다.
