# Step 1 실험 결과 및 분석

이 문서는 좌표 불변 GRU 베이스라인 모델의 리더보드 성능 결과와 그에 대한 분석을 기록합니다.

## 1. 리더보드 성능 비교

| 모델명 | 리더보드 점수 (Private) | 비고 |
| :--- | :--- | :--- |
| **기존 Baseline (CV)** | **0.6006** | 등속 외삽(Constant Velocity) 기반 |
| **Baseline GRU (Step 1)** | **0.5374** | 상대 좌표 기반 단순 GRU |

*분석: 기존의 물리 법칙 기반 등속 모델에 비해 딥러닝 베이스라인의 성능이 낮게 측정됨.*

---

## 2. 성능 저하 원인 분석

### 2.1. "평균의 함정" (Averaging Problem)
*   모기의 비행은 다중 모드(Multimodal) 특성을 가집니다. 즉, 같은 과거 경로에서도 미래에는 왼쪽, 오른쪽, 혹은 위로 꺾는 다양한 가능성이 존재합니다.
*   단순 GRU 모델은 MSE(평균 제곱 오차)를 최소화하도록 학습되는데, 이는 모든 가능한 미래 위치의 **'산술 평균'**을 예측하게 만듭니다.
*   이로 인해 모델의 예측값이 역동성을 잃고 중심부로 수렴(Blurry prediction)하게 되어, 물리 법칙을 충실히 따르는 등속 모델보다 오차가 커졌습니다.

### 2.2. 물리적 제약 조건(Prior)의 부재
*   등속 모델은 "물체는 진행 방향으로 계속 이동한다"는 강력한 물리적 가정(Prior)을 가지고 있습니다.
*   반면 Step 1 모델은 순수하게 데이터의 통계적 패턴만 학습하므로, 짧은 관측 기간(400ms) 내의 노이즈에 민감하게 반응하거나 물리적으로 타당하지 않은 궤적을 생성할 수 있습니다.

### 2.3. 데이터 증강 및 정교함 부족
*   단일 궤적 데이터만으로는 모델이 일반화된 비행 규칙을 배우기에 부족했을 수 있습니다.
*   회전 변환(Rotation) 등 기하학적 특성을 고려한 데이터 증강이 적용되지 않아 다양한 각도의 비행에 대응력이 떨어졌습니다.

---

## 3. 개선 방향 (Step 2의 목적)
Step 1의 한계를 극복하기 위해 다음 단계에서는 **다중 모드 목표 지점(Multimodal Goal Anchors)** 방식을 도입합니다.
1.  하나의 평균 지점이 아닌, 6개의 대표 비행 앵커를 제안하여 '의도'를 먼저 파악합니다.
2.  의도가 파악된 상태에서 미세 조정(Offset)을 수행하여 '평균의 함정'을 탈출합니다.
3.  이를 통해 등속 모델이 잡지 못하는 급격한 기동(Maneuver) 구간에서의 성능 향상을 목표로 합니다.
